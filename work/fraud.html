Credit Card Fraud Detection on a 0.17% Positive Class

European Cardholders • Public Dataset

Fraud detection is a textbook imbalanced classification problem: 99.83% of transactions are legitimate, and a model that simply predicts "not fraud" for everything will hit 99.83% accuracy while missing every fraud. This project explores how under-sampling and a calibrated logistic regression baseline can do meaningful work on a stubborn dataset.

The Core Challenge

Why accuracy is the wrong metric

The dataset contains roughly 284,000 transactions and only 492 are fraud. A naive classifier that always says "legitimate" would still hit 99.83% accuracy. That's the trap: the metric looks great while the model is useless.

The real signal lives in precision and recall on the minority class. The work isn't building a fancier model — it's reshaping the training data so the model is forced to see fraud often enough to learn its structure.

Approach

Under-sampling, not magic

  • Exploratory profiling. Compared mean transaction amounts and feature distributions across the legitimate and fraudulent classes to confirm there is real signal to learn (fraud transactions tend to cluster differently in the PCA-derived feature space).
  • Under-sampling. Built a balanced training sample by drawing 492 legitimate transactions to match the 492 fraud cases. This gives the model an even diet during training without synthetic data risks.
  • Baseline classifier. Trained a logistic regression on the balanced sample. Logistic regression is the right starting point: interpretable coefficients, fast to train, easy to calibrate.
  • Held-out evaluation. Tested on the original imbalanced distribution to see how the under-sampled model generalises to the real world.
What I Learned

The bias-variance trap of imbalance

Under-sampling solves the training problem but introduces its own one: by throwing away legitimate transactions, you lose information about what "normal" looks like. The right next step would be to compare under-sampling against SMOTE (synthetic minority over-sampling) and class-weighted loss functions, then evaluate all three on precision-recall AUC rather than accuracy.

Working on this dataset hammered in a lesson that no textbook example ever quite teaches: the metric you optimise for shapes the model you get. Pick the wrong metric and you'll ship a model that brags about accuracy while missing every fraud.
